01

Nền tảng lý thuyết — Phải hiểu điều này trước

Trước khi đi vào từng method, cần nắm ba khái niệm nền tảng của Functional Programming. Hiểu đúng ba khái niệm này giúp bạn sử dụng các array methods một cách có chủ đích, tránh bug khó chịu.

Higher-Order Function Pure Function Immutability Functional Programming
Higher-Order Function
Hàm nhận một hàm khác làm tham số, hoặc trả về một hàm. map, filter, reduce đều là HOF — chúng nhận callback function và áp dụng lên từng phần tử.
Pure Function
Hàm không có side effect — cùng input luôn cho cùng output, không thay đổi biến bên ngoài. Callback trong map/filter/reduce nên là pure function.
Immutability
map và filter không thay đổi mảng gốc mà trả về mảng mới. Nguyên tắc này giúp code dễ debug và tránh bug do mutation.
💡
Tại sao quan trọng? Trong React, việc mutation state trực tiếp là bug kinh điển. Hiểu immutability giúp bạn dùng map để update, filter để delete items trong state array một cách an toàn.
02

Array.prototype.map() — Biến đổi từng phần tử

Định nghĩa chính xác: map tạo ra một mảng mới bằng cách áp dụng hàm callback lên từng phần tử của mảng gốc. Kết quả là mảng mới có cùng độ dài với mảng ban đầu.

Signature đầy đủ
array.map(callback(currentValue, index, array), thisArg)
javascript 
// Ví dụ cơ bản
const prices = [100, 200, 350, 80];
const withTax = prices.map(price => price * 1.1);
// [110, 220, 385, 88] — mảng gốc prices không đổi

// Dùng đủ 3 tham số của callback
const labeled = prices.map((price, index, arr) => ({
  id: index + 1,
  price,
  isMax: price === Math.max(...arr)
}));

// Thực tế: transform API response
const users = apiData.map(user => ({
  id: user.user_id,          // đổi tên key
  name: `${user.first_name} ${user.last_name}`, // kết hợp fields
  isAdmin: user.role === 'admin' // tính toán derived value
}));

Hiểu sâu — Cách map() hoạt động bên trong

javascript — polyfill 
// Polyfill đơn giản giúp bạn hiểu cơ chế
Array.prototype.myMap = function(callback, thisArg) {
  const result = new Array(this.length); // tạo mảng cùng size
  for (let i = 0; i < this.length; i++) {
    if (i in this) { // bỏ qua "holes" trong sparse array
      result[i] = callback.call(thisArg, this[i], i, this);
    }
  }
  return result;
};

Sparse Array — Chi tiết quan trọng

javascript 
// map giữ nguyên "holes" (phần tử trống)
const sparse = [1, , 3]; // sparse array, index 1 bị bỏ qua
sparse.map(x => x * 2); // [2, empty, 6] — không phải [2, NaN, 6]

// Tạo array range với Array.from + map
Array.from({ length: 10 }, (_, i) => i); // [0, 1, 2, ..., 9]

// Fill + map để tạo range
Array(3).fill(0).map((_, i) => i); // [0, 1, 2]
03

Array.prototype.filter() — Lọc phần tử

Định nghĩa chính xác: filter tạo mảng mới chứa chỉ những phần tử mà callback trả về truthy. Mảng kết quả có độ dài bằng hoặc nhỏ hơn mảng gốc.

Signature đầy đủ
array.filter(callback(currentValue, index, array), thisArg)
javascript — ví dụ cơ bản 
// Ví dụ cơ bản
const scores = [88, 45, 92, 60, 78, 30];
const passed = scores.filter(score => score >= 60);
// [88, 92, 60, 78]

// Lọc object phức tạp
const products = [
  { name: 'Laptop', price: 1200, inStock: true },
  { name: 'Mouse', price: 25, inStock: false },
  { name: 'Monitor', price: 400, inStock: true },
];

const affordable = products.filter(p => p.price < 500 && p.inStock);
// [{ name: 'Monitor', price: 400, inStock: true }]
javascript — patterns hay 
// Pattern: Loại bỏ falsy values (null, undefined, 0, '', false)
const messy = [1, null, 2, undefined, 3, '', 4, false];
const clean = messy.filter(Boolean); // [1, 2, 3, 4]

// Pattern: Loại bỏ duplicates (kết hợp với indexOf)
const nums = [1, 2, 2, 3, 3, 3, 4];
const unique = nums.filter((val, idx, arr) => arr.indexOf(val) === idx);
// [1, 2, 3, 4] — chỉ giữ lần xuất hiện đầu tiên

// Lưu ý: với dữ liệu lớn dùng Set hiệu quả hơn
const uniqueFast = [...new Set(nums)];

TypeScript — Type Predicate với filter()

Đây là kiến thức quan trọng khi làm việc với TypeScript. filter không tự động narrow type — bạn cần dùng type predicate.

typescript 
const mixed: (string | null)[] = ['a', null, 'b', null, 'c'];

// ❌ Sai: TypeScript vẫn thấy (string | null)[]
const wrong = mixed.filter(x => x !== null);

// ✅ Đúng: dùng type predicate — "(x): x is string"
const correct = mixed.filter((x): x is string => x !== null);
// Type của correct là string[] ✓

// Type predicate tổng quát — reusable
const isNonNull = <T>(value: T | null | undefined): value is T => value != null;
const result = mixed.filter(isNonNull); // string[]

Array.prototype.reduce() — Hàm mạnh nhất

reduce là "superpower" của array methods. Nó có thể triển khai lại toàn bộ map, filter, và hầu hết mọi array operation khác. Accumulator là giá trị tích lũy qua từng bước iteration.

Signature đầy đủ
array.reduce(callback(accumulator, currentValue, index, array), initialValue)
⚡
initialValue cực kỳ quan trọng! Nếu không có initialValue và mảng rỗng → TypeError ngay lập tức. Luôn cung cấp initialValue là thói quen tốt.
javascript — use cases thực tế 
// 1. Tính tổng / tích lũy số học
const sum = [1, 2, 3, 4, 5].reduce((acc, val) => acc + val, 0); // 15

// 2. Tìm max/min (không dùng Math.max)
const max = [3, 1, 4, 1, 5, 9].reduce((acc, val) => val > acc ? val : acc);

// 3. Group by — cực phổ biến trong thực tế
const students = [
  { name: 'An', grade: 'A' },
  { name: 'Bình', grade: 'B' },
  { name: 'Châu', grade: 'A' },
  { name: 'Dũng', grade: 'B' },
  { name: 'Em', grade: 'C' },
];

const grouped = students.reduce((acc, student) => {
  const key = student.grade;
  if (!acc[key]) acc[key] = [];
  acc[key].push(student);
  return acc;
}, {});
// { A: [{name:'An',...}, {name:'Châu',...}], B: [...], C: [...] }

// 4. Đếm tần suất (frequency count)
const words = ['apple', 'banana', 'apple', 'cherry', 'banana', 'apple'];
const freq = words.reduce((acc, word) => {
  acc[word] = (acc[word] || 0) + 1;
  return acc;
}, {});
// { apple: 3, banana: 2, cherry: 1 }

// 5. Flatten mảng lồng nhau
const nested = [[1, 2], [3, 4], [5, 6]];
const flat = nested.reduce((acc, arr) => acc.concat(arr), []);
// [1, 2, 3, 4, 5, 6]
javascript — reduce implement lại map và filter 
// Implement map bằng reduce (chứng minh sức mạnh)
const doubled = [1, 2, 3].reduce((acc, val) => {
  acc.push(val * 2);
  return acc;
}, []);

// Implement filter bằng reduce
const evens = [1, 2, 3, 4, 5, 6].reduce((acc, val) => {
  if (val % 2 === 0) acc.push(val);
  return acc;
}, []);

// Pipe / compose functions — functional programming pattern
const pipe = (...fns) => (x) => fns.reduce((v, fn) => fn(v), x);

const transform = pipe(
  x => x * 2,
  x => x + 10,
  x => `$${x}`
);
transform(5); // '$20'

⚠️ Cạm bẫy nguy hiểm — Bắt buộc phải biết

javascript — common pitfalls 
// ⚠️ NGUY HIỂM: Không có initialValue với mảng rỗng
[].reduce((acc, val) => acc + val);
// → TypeError: Reduce of empty array with no initial value

// ✅ AN TOÀN: Luôn cung cấp initialValue
[].reduce((acc, val) => acc + val, 0); // 0 — an toàn

// ❌ LỖI THƯỜNG GẶP: Quên return accumulator
const wrong = [1, 2, 3].reduce((acc, val) => {
  acc.total += val; // ← Không có return!
}, { total: 0 });
// wrong === undefined ← sai!

// ✅ ĐÚNG: Phải return accumulator
const correct = [1, 2, 3].reduce((acc, val) => {
  acc.total += val;
  return acc; // ← Bắt buộc!
}, { total: 0 });

Method Chaining — Sức mạnh thực sự

Vì mỗi method trả về mảng mới, bạn có thể chain liên tiếp, tạo ra một pipeline xử lý dữ liệu cực kỳ dễ đọc và maintain. Đây là cách code thực tế trong production.

javascript — chaining pipeline 
const orders = [
  { id: 1, customer: 'An', amount: 150, status: 'completed', category: 'electronics' },
  { id: 2, customer: 'Bình', amount: 50, status: 'pending', category: 'books' },
  { id: 3, customer: 'Châu', amount: 320, status: 'completed', category: 'electronics' },
  { id: 4, customer: 'Dũng', amount: 80, status: 'cancelled', category: 'books' },
  { id: 5, customer: 'Em', amount: 200, status: 'completed', category: 'electronics' },
];

// Tổng doanh thu từ đơn electronics đã completed
const revenue = orders
  .filter(order => order.status === 'completed')    // [1, 3, 5]
  .filter(order => order.category === 'electronics') // [1, 3, 5]
  .map(order => order.amount)                        // [150, 320, 200]
  .reduce((sum, amount) => sum + amount, 0);        // 670

// Danh sách tên khách hàng unique đã hoàn thành đơn
const completedCustomers = orders
  .filter(o => o.status === 'completed')
  .map(o => o.customer)
  .filter((name, idx, arr) => arr.indexOf(name) === idx);

Performance — Khi nào KHÔNG nên chain

javascript — performance optimization 
// ⚠️ Chaining duyệt mảng 3 lần
const result = bigArray
  .filter(x => x > 0)    // pass 1: O(n)
  .map(x => x * 2)       // pass 2: O(n)
  .reduce((a, b) => a + b, 0); // pass 3: O(n)

// ✅ Tối ưu: dùng reduce một lần — O(n) duy nhất
const result = bigArray.reduce((acc, x) => {
  if (x > 0) acc += x * 2;
  return acc;
}, 0);

// Ghi nhớ: với dữ liệu nhỏ-vừa (<100k phần tử),
// chaining vẫn ổn — ưu tiên readability hơn.

Patterns nâng cao quan trọng

Pattern 1: flatMap() — ES2019

flatMap là sự kết hợp của map + flat(1) trong một bước. Rất hữu ích khi một phần tử cần được "expand" thành nhiều phần tử.

javascript — flatMap 
// map thông thường tạo mảng lồng nhau
const sentences = ['hello world', 'foo bar'];
sentences.map(s => s.split(' ')); // [['hello', 'world'], ['foo', 'bar']]

// flatMap tự động flatten 1 cấp
sentences.flatMap(s => s.split(' ')); // ['hello', 'world', 'foo', 'bar']

// Thực tế: expand một item thành nhiều items
const cart = [
  { product: 'T-shirt', qty: 2 },
  { product: 'Shoes', qty: 1 },
];
const items = cart.flatMap(item =>
  Array(item.qty).fill(item.product)
);
// ['T-shirt', 'T-shirt', 'Shoes']

Pattern 2: Xử lý bất đồng bộ (async/await)

Đây là lỗi cực kỳ phổ biến của developer khi dùng map với async. Cần đặc biệt chú ý!

javascript — async với map 
// ❌ SAI: map với async không chờ Promise!
const results = await arr.map(async item => await fetch(item.url));
// results là mảng Promise[], không phải kết quả!

// ✅ ĐÚNG: Parallel — dùng Promise.all + map
const results = await Promise.all(
  arr.map(async item => {
    const res = await fetch(item.url);
    return res.json();
  })
);

// ✅ Sequential (tuần tự) — dùng reduce với async
const results = await arr.reduce(async (prevPromise, item) => {
  const acc = await prevPromise;
  const result = await fetch(item.url).then(r => r.json());
  return [...acc, result];
}, Promise.resolve([]));
// Mỗi request chờ request trước hoàn thành

Pattern 3: Object.entries() + reduce()

Combo này cực kỳ hay khi làm việc với objects — transform keys, filter properties, hay parse types.

javascript — object transformation 
const config = { host: 'localhost', port: '3000', debug: 'true' };

// Chuyển string → đúng type (auto-parse)
const parsed = Object.entries(config).reduce((acc, [key, val]) => {
  const num = Number(val);
  acc[key] = !isNaN(num) ? num        // số
           : val === 'true' ? true    // boolean true
           : val === 'false' ? false  // boolean false
           : val;                     // giữ nguyên string
  return acc;
}, {});
// { host: 'localhost', port: 3000, debug: true }

// Lọc keys của object (chỉ giữ public keys)
const publicConfig = Object.entries(config)
  .filter(([key]) => !key.startsWith('_'))
  .reduce((acc, [key, val]) => ({ ...acc, [key]: val }), {});

Pattern 4: reduceRight() — Compose functions

javascript — compose vs pipe 
// pipe: thực hiện từ trái sang phải (dùng reduce)
const pipe = (...fns) => (x) => fns.reduce((v, fn) => fn(v), x);

// compose: thực hiện từ phải sang trái (dùng reduceRight)
const compose = (...fns) => (x) => fns.reduceRight((v, fn) => fn(v), x);

const process = compose(
  x => `Result: ${x}`,  // bước 3
  x => x * 2,            // bước 2
  x => x + 5             // bước 1
);
process(3); // 'Result: 16' — 3+5=8 → 8×2=16 → 'Result: 16'

Kiến thức liên quan quan trọng

Array.from() — Tạo mảng từ iterable

javascript — Array.from 
// Tạo range [0..9] — pattern rất hay
Array.from({ length: 10 }, (_, i) => i); // [0, 1, 2, ..., 9]

// Tạo range với giá trị cụ thể
Array.from({ length: 5 }, (_, i) => i * 2); // [0, 2, 4, 6, 8]

// Convert NodeList (DOM) sang array để dùng map/filter
const links = Array.from(document.querySelectorAll('a'))
  .map(a => a.href)
  .filter(href => href.startsWith('https'));

// Spread cũng work cho iterables
const chars = [...'hello'].map(c => c.toUpperCase()).join(''); // 'HELLO'

Immutable State trong React — Pattern thực tế

javascript — React/Redux immutable patterns 
// Pattern UPDATE: cập nhật item không mutation
const updateUser = (users, id, newData) =>
  users.map(user => user.id === id ? { ...user, ...newData } : user);

// Pattern DELETE: xóa item
const removeUser = (users, id) =>
  users.filter(user => user.id !== id);

// Pattern ADD: thêm item
const addUser = (users, newUser) => [...users, newUser];

// React useState example — đúng cách
const [todos, setTodos] = useState([...]);

// Toggle done
const toggleTodo = (id) =>
  setTodos(prev => prev.map(t => t.id === id ? { ...t, done: !t.done } : t));

// Delete todo
const deleteTodo = (id) =>
  setTodos(prev => prev.filter(t => t.id !== id));

So sánh với forEach — Khi nào KHÔNG dùng map

javascript — map vs forEach 
// forEach: khi cần side effects, không cần return value
orders.forEach(order => {
  console.log(order.id);     // side effect: log
  sendEmail(order.customer); // side effect: API call
});

// ❌ SAI: dùng map chỉ cho side effects — lãng phí
orders.map(order => console.log(order.id));
// Tạo mảng [undefined, undefined, ...] không ai dùng

// ✅ ĐÚNG: map khi thực sự cần mảng mới
const ids = orders.map(order => order.id);

// Sự khác biệt quan trọng:
// forEach → trả về undefined, không chain được
// map/filter/reduce → trả về giá trị, chain được

Tóm tắt & Bảng quyết định

Câu hỏi cần tự hỏi khi xử lý mảng:

Câu hỏi bạn tự hỏi Method dùng Ví dụ nhanh
Muốn biến đổi từng phần tử, giữ số lượng? .map() prices → prices with tax
Chỉ lấy những phần tử thỏa điều kiện? .filter() users → active users only
Gộp tất cả thành một giá trị hoặc object? .reduce() items → total, group by, freq
Biến đổi VÀ làm phẳng mảng lồng? .flatMap() sentences → words
Chỉ cần side effects, không cần kết quả? .forEach() log, API calls, DOM updates
Kiểm tra TẤT CẢ phần tử thỏa điều kiện? .every() all scores >= 50?
Kiểm tra ÍT NHẤT MỘT phần tử thỏa điều kiện? .some() any score < 50?
Tìm phần tử đầu tiên thỏa điều kiện? .find() first user with role=admin

Mental Model — Hình dung dữ liệu chảy qua pipeline

javascript — mental model 
// Hình dung: data pipeline như assembly line
const result = rawData
  .filter(isValid)          // 🚧 QC gate: loại hàng lỗi
  .map(normalize)           // 🔧 Transform: chuẩn hóa format
  .filter(meetsRequirement) // 🚧 QC gate 2: kiểm tra yêu cầu
  .map(enrich)              // 🔧 Transform: bổ sung thêm data
  .reduce(aggregate, {});   // 📦 Pack: đóng gói kết quả

// 3 phương thức này là backbone của data transformation
// trong JavaScript hiện đại — React, Node.js, hay bất kỳ
// framework nào đều dùng chúng hàng ngày.
✅
Key takeaways để nhớ mãi:
• map — "tôi muốn mảng khác với cùng số phần tử"
• filter — "tôi muốn subset của mảng này"
• reduce — "tôi muốn chưng cất mảng này thành một thứ"
• Luôn cung cấp initialValue cho reduce
• Không dùng map chỉ cho side effects — đó là việc của forEach
• map/filter/reduce không mutation mảng gốc — đây là immutability
